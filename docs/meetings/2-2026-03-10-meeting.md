## Meeting – 2026-02-12

### Discussed Points
- I explained how I developed the `data fetcher` and the overall setup.
- Initially I was using `Etherscan APIs` to fetch transaction data, but the rate limits were quite restrictive.
- Because of that I migrated the fetching process to `Infura`, which allows better scaling and fewer limitations.
- The fetched data is now stored in a `PostgreSQL` database.
- I also implemented a `CLI tool` to manage the fetching process. The idea is that if needed, it can later be deployed on `AWS` or another server to continuously collect data.
- I explained which `networks` and `stablecoins` I selected and the reasoning behind those choices.
- We also briefly discussed whether it would be possible to integrate the `x402 protocol`.
- While searching online, we found `https://www.8004scan.io/` which lists some agents.
- The idea is that these could potentially be mapped to the `from` and `to` fields of transactions to detect if a transaction was done by an agent.
